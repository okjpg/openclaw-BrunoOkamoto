3
Qual Modelo Para Cada Situação

A estratégia: use o modelo mínimo que resolve o problema.

Não use GPT-5.4 pra checar o clima. Não use GPT-4o-mini para uma decisão crítica de negócio.

⏰ Heartbeats e Crons
→ GPT-4o-mini
Heartbeat é quando o agente acorda sozinho pra checar emails, calendário, notificações. Tarefas simples: verificar, categorizar, decidir se precisa te avisar. GPT-4o-mini faz isso com excelência e custo mínimo.
💰 Heartbeats e crons são o lugar ideal para GPT-4o-mini. Reserve a verba pesada para quando você realmente precisar de contexto longo ou análise profunda.
💬 Interação Diária (suas mensagens no Telegram)
→ GPT-4o
Quando você manda uma mensagem pro agente e quer uma resposta útil — pesquisa, organização, análise leve, escrita. GPT-4o é rápido, confiável e é o melhor ponto da stack padrão para interação diária.
🎯 O ponto ideal da curva preço × qualidade
🧠 Análise Complexa e Decisões Críticas
→ GPT-5.4
Documento longo, decisão arquitetural, proposta comercial, análise sensível. GPT-5.4 quando você precisa do melhor raciocínio da stack padrão — mas só nesses momentos.
⚠️ Reserve para quando realmente importa
💡 Alternativas Econômicas
→ Gemini 3.1 Pro ou Mistral Small
Gemini 3.1 Pro ($1.25/M) — excelente alternativa ao Sonnet pra quem quer economizar. Funciona muito bem desde a v2026.3.2.

Mistral Small ($0.10/M) — absurdamente barato, ideal para classificação simples e routing. Não use para raciocínio complexo.
4
Como Configurar Modelo por Função

O OpenClaw permite modelos diferentes por função.

Você pode ter GPT-4o-mini para heartbeats, GPT-4o para conversas e GPT-5.4 reservado para análises pesadas. Esse é o stack padrão do curso.

# Modelo padrão para interações do dia a dia
openclaw config set model openai/gpt-5.4

# Modelo econômico para heartbeats (automático)
openclaw config set heartbeat.model openai/gpt-4o-mini

# Verificar o que está configurado agora
openclaw config get model
openclaw config get heartbeat.model
🔀
Funciona automaticamente

Depois de configurar, o agente usa Haiku nos heartbeats e Sonnet quando você manda mensagem — sem você precisar especificar nada. Quer mudar de modelo a qualquer hora? openclaw config set model NOVO-MODELO e pronto, vale a partir da próxima mensagem.

Proteção — "Minha Conta Zerou"

Configure os limites antes de usar intensamente.

Um loop mal configurado pode fazer milhares de chamadas por engano. Esses são seus cintos de segurança.

⚠️
Faça isso agora, antes de continuar

platform.openai.com → Settings → Billing → Usage Limits → defina um "Monthly spend limit" (ex: $50) e um "Notification threshold" (ex: $25). Dois minutos que podem economizar centenas de dólares.

No platform.openai.com:

Settings → Billing → Usage Limits
Monthly spend limit: $50
Notification threshold: $25

No OpenClaw:

# Limite de requests/hora
openclaw config set \
  rateLimit.requestsPerHour 100

# Ver gasto do mês
openclaw usage report

Dúvidas Frequentes
Posso trocar de modelo a qualquer hora?
Sim. openclaw config set model NOVO-MODELO e pronto. Válido a partir da próxima mensagem. Não precisa reiniciar nada.
O Mistral é muito mais barato — por que não usar sempre?
Qualidade inferior para raciocínio complexo. Para heartbeats simples funciona; para conversas e análises, a perda de qualidade é perceptível. Use Haiku como mínimo para interações com o agente.
Gemini é confiável para uso com OpenClaw?
Sim, desde a v2026.3.2. Boa alternativa econômica ao Sonnet. Para casos muito complexos com ferramentas encadeadas, Claude ainda leva vantagem — mas para uso geral é excelente.
Haiku é "burro"?
Não! Para tarefas estruturadas — checar emails, classificar mensagens, responder perguntas diretas — Haiku é excelente. Fica atrás em raciocínio multi-step e criatividade. Para 80% das tarefas de automação, é mais que suficiente.
Como saber exatamente quanto gastei?
openclaw usage report mostra o breakdown por modelo. O platform.openai.com mostra em tempo real com gráficos e histórico por dia.
